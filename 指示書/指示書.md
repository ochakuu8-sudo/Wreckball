# Wreckball Rampage 正式統合 実装指示書

## 1. 実装目的

現在の実装は、`game-rampage.ts` が既存 `Game` に runtime patch を当ててRampage系ルールを差し込んでいる暫定版である。

この方式は検証には使えるが、正式実装としては不安定である。

主な問題:

- 旧 `fuel` 制と新 `SPEED / momentum` 制が混ざる
- `stage_clear` / `clear` / castle goal など旧ステージ制が残る
- 人間クラッシュ報酬を推測で検出しており、`+SPEED` 表示の暴発や取り逃がしが起きる
- 建物破壊時の人間バーストが `HumanManager` の内部構造推測に依存している
- 道路・セル・ステージ生成と、ランダム建物散布が混在している
- どれが本体なのか分かりにくい

このため、最終目的は以下である。

```txt
旧ゲームの見た目・物理・描画品質を維持したまま、
ゲームルールをRampage仕様へ正式に置き換える。
```

最終的な起動経路は以下に戻す。

```ts
// src/main.ts
import { Game } from './game';
```

そのうえで、`src/game.ts` 自体をRampage仕様の正式本体にする。

## 2. 完成時のゲーム体験

目指すプレイループ:

```txt
小型建物が多めにランダム散布されている
↓
ボールで建物を壊す
↓
建物から人間が飛び出す
↓
人間を潰す
↓
SPEEDが上がる
↓
SPEEDが高いほど中型・大型建物も壊しやすくなる
↓
CHAINが伸びる
↓
OVERDRIVEで破壊効率が上がる
↓
SPEEDが低い状態が続くとRUN OVER
```

このゲームは「硬い建物を攻略するゲーム」ではなく、**小型建物を気持ちよく連続破壊してSPEEDを維持するゲーム**として設計する。

## 6. Rampageルール設計

### 6.1 SPEED / momentum

旧 `fuel` の代わりに `momentum` を使う。
プレイヤー向け表示は `SPEED` とする。

```txt
momentum は時間で自然減衰する
人間を潰すと momentum が上がる
momentum が高いほど camera.scrollSpeed が上がる
momentum が低い状態が続くと RUN OVER
```

### 6.2 CHAIN

人間を連続で潰すと `combo / chain` が伸びる。

```txt
人間を潰す
↓
CHAIN +1
↓
短時間以内に次の人間を潰す
↓
CHAIN継続
↓
SPEED上昇量が増える
```

建物破壊はCHAINを直接増やさない。建物は人間を出すためのトリガー。

### 6.3 OVERDRIVE

一定CHAIN以上でOVERDRIVEに入る。

効果:

```txt
- momentum減少軽減
- 建物へのダメージ増加
- 破壊演出強化
```

Powerゲートではなく、破壊効率アップとして扱う。

## 7. 建物仕様

### 7.1 Powerゲートは廃止

以前の案では `Wreck Power I〜V` によって壊せる建物を制限していたが、これは廃止する。

理由:

```txt
速度が低いと壊せない
```

より、

```txt
速度が低くても壊せるが、速度が高いほど壊しやすい
```

方がこのゲームに合っているため。

### 7.2 建物はHP制

全建物はHPを持つ。
小型建物は基本一撃破壊。
中型以上はSPEEDに応じてダメージが増える。

```txt
小型建物: 常に一撃
中型建物: 低速なら1〜2発、高速なら一撃寄り
大型建物: 低速でも削れる、高速で壊しやすい
特大建物: レア、OVERDRIVEで壊しやすい
```

### 7.3 建物出現比率

初期バランス:

```txt
小型: 80%
中型: 15%
大型: 4%
特大: 1%
```

### 7.4 建物クラス

`src/rampage-building-profiles.ts` で分類する。

```ts
export type RampageBuildingClass = 'small' | 'medium' | 'large' | 'huge';
```

小型は主食。低SPEEDからの復帰手段でもある。

## 8. 建物生成

道路・セル生成をやめ、スクロール方向のbandに建物をランダム散布する。

新規ファイル:

```txt
src/rampage-generation.ts
```

### 8.1 API案

```ts
export interface BuildingSpawnDef {
  x: number;
  y: number;
  size: C.BuildingSize;
  blockIdx?: number;
}

export function generateInitialRampageCity(): BuildingSpawnDef[];

export function generateRampageBand(
  baseY: number,
  bandHeight: number,
  blockIdx: number
): BuildingSpawnDef[];
```

### 8.2 配置ルール

```txt
1. 建物種別を重み付きで選ぶ
2. x/yをランダム決定
3. AABB + padding で重なり判定
4. 既存候補と重ならなければ採用
5. 60回失敗したらその候補は諦める
```

### 8.3 安全制約

```txt
- フリッパー付近を塞がない
- 中央ドレインを塞がない
- 特大建物を連続配置しない
- 初期画面には小型建物を必ず複数置く
- 建物同士を重ねない
```

## 9. 人間システム

現在の暫定実装では、人間バーストやクラッシュ報酬をruntime patchで推測している。
正式実装では `HumanManager` に明示的なAPIを追加する。

対象:

```txt
src/humans.ts
```

### 9.1 建物破壊時の人間バースト

追加メソッド案:

```ts
spawnPanicHumansFromBuilding(
  building: BuildingData,
  count: number,
  value: number
): void
```

挙動:

```txt
- 建物中心付近から出る
- mode = FREE / panic
- 建物中心から外向きに逃げる
- life = 3.5〜6秒
- 建物タイプに応じた人間バリエーションを使う
```

### 9.2 人間クラッシュイベント

`+SPEED`暴発を防ぐため、Game側が正確に人間クラッシュを拾えるイベントを追加する。

```ts
export interface HumanCrushEvent {
  x: number;
  y: number;
  value: number;
}

consumeCrushEvents(): HumanCrushEvent[]
```

人間が潰された瞬間に `HumanManager` 内部へイベントを積み、`Game.update()` が毎フレームそれを消費する。

## 12. 建物衝突処理

Powerゲートは使わない。

建物に当たったら、以下を行う。

```ts
const profile = getRampageBuildingProfile(bld.size);

const damage = profile.guaranteedOneShot
  ? bld.hp
  : computeRampageDamage(
      bld.size,
      this.rampage.momentum,
      this.rampage.isOverdrive()
    );

const destroyed = this.buildings.damage(bld, damage);
```

破壊された場合:

```ts
this.onBuildingDestroyed(bld);
```

## 13. 建物破壊処理

`game.ts` に正式メソッドを追加する。

```ts
private onBuildingDestroyed(bld: BuildingData): void {
  const profile = getRampageBuildingProfile(bld.size);

  const raw = randInt(bld.humanMin, bld.humanMax);
  const speedBonus = 1 + Math.min(1, this.rampage.momentum / 100) * 0.35;
  const count = clampHumanBurst(
    bld.size,
    Math.floor(raw * 0.25 * speedBonus)
  );

  this.humans.spawnPanicHumansFromBuilding(
    bld,
    count,
    this.humanValueForBuilding(bld)
  );

  this.rampage.onBuildingDestroyed(profile.klass);
}
```

## 14. 人間報酬処理

`Game.update()` 内で以下を行う。

```ts
for (const ev of this.humans.consumeCrushEvents()) {
  const reward = this.rampage.onHumanCrushed(ev.value);

  this.totalHumans++;
  this.totalScore += this.scoreForHuman(reward.combo);

  this.ui.showSpeedPopup(
    ev.x,
    ev.y,
    reward.combo,
    reward.overdriveStarted
  );
}
```

これにより、`+SPEED` は実際に人間を潰した時だけ出る。

## 15. RUN OVER

低SPEEDが一定時間続いたら終了。

```ts
if (this.rampage.updateFailureTimer(dt)) {
  this.state = 'game_over';
  gameplayStop();
  this.ui.showRunOver(...);
}
```

即死ではなく、復帰猶予を残す。

## 20. 注意点

最重要注意点は、旧処理と新処理を二重に走らせないこと。

特に以下は必ず整理する。

```txt
- fuel drain
- fuel回復
- stage clear
- clear
- 建物生成
- 人間報酬
- UI更新
```

これが二重になると、以下のようなバグが再発する。

```txt
- +SPEEDが暴発する
- SPEEDとfuelが混ざる
- STAGE表示が残る
- 想定外の建物生成が残る
- 人間バーストが二重になる
```
